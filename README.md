# AI-ENABLED-MILITARY-MEDICAL-PRIORITIZATION-SYSTEM-FOR-CRITICAL-BATTLEFIELD-CARE

An AI-driven system designed to assist in prioritizing injured soldiers and supporting medics with real-time medical decisions in critical environments.

---

## Overview

In high-pressure situations such as battlefields, timely and accurate decisions play a crucial role in saving lives. Traditional triage methods rely heavily on manual judgment, which can become slow and inconsistent when dealing with multiple casualties.

This project introduces a machine learning-based approach to improve the efficiency of medical response. By analyzing patient vitals and conditions, the system classifies the severity level and provides basic first-aid guidance. The aim is to support medics in making faster, more informed decisions during emergency scenarios.

---

## Project Status

⚠️ This project is currently under development and is in the prototype stage.

While the core functionalities such as severity prediction and basic decision support are implemented, the system is not yet production-ready. Further improvements are being made to enhance accuracy, scalability, and real-time integration.

---

## Key Features

-  **Severity Prediction** using machine learning models  
-  **First-Aid Suggestions** based on patient condition  
-  **Multi-Patient Handling** for better prioritization  
-  **Alert System** for identifying critical cases

## Datasets

This project uses publicly available medical datasets:

- PhysioNet (physiological signals like heart rate, BP, SpO₂)  
- MIMIC-III / MIMIC-IV (clinical patient data)  

Due to size and access restrictions, datasets are not included in this repository.

- PhysioNet: https://physionet.org/  
- MIMIC Dataset: https://physionet.org/content/mimiciv/

## Objective

The primary objective of this project is to assist in faster and more reliable medical decision-making during emergency situations. By reducing delays and supporting prioritization, the system aims to improve response efficiency, especially during the critical “golden hour.”

### Guidance Window:

<img width="1170" height="583" alt="image" src="https://github.com/user-attachments/assets/4b9e7091-08ba-468d-911b-ba57b79d7c50" />

This panel is the primary decision-support display for the field medic. It uses a read-only interface to present real-time, prioritized triage data derived from the continuous AI monitoring of all casualties. Its function is to rapidly communicate the Highest Priority Patient's Status (color-coded,high-contrast severity tags), the relevant Fuzed Vitals used for classification, and the Actionable Recommended Actions (step-by-step first-aid guidance) generated by the system's neural network logic.

### Casualty Status Panel:

<img width="230" height="550" alt="image" src="https://github.com/user-attachments/assets/22cbf631-f034-4486-bba4-eb502042abc4" />

This panel provides the Multi-Casualty Analysis overview, listing the current triage status of every known patient being monitored by the system (a core requirement of the project). It functions as a scrollable, read-only list that allows medics or command staff to quickly scan the status of all available data streams. Patients are typically sorted by severity (Critical at the top) and the display uses high-contrast color coding to instantly distinguish between Critical, Warning, and Normal cases, ensuring no casualty is missed, even if they are not currently the absolute highest priority.

### Alerts Log:

<img width="1725" height="232" alt="image" src="https://github.com/user-attachments/assets/e1f3c6ab-2fc3-4cac-9e3b-340504092261" />

This panel is the central, thread-safe console for the entire Military Triage System. It provides essential historical logging for all application events, enabling auditability and real-time awareness for both the field medic and remote headquarters (HQ).The log tracks three main types of events:
- SYSTEM: Records internal application status changes, such as the camera initialization, switching between live feed patients (Multi-Casualty analysis), and system shutdowns.
- PRIORITY: Logs any significant shift in triage, specifically indicating when the system has prioritized a different casualty for display on the main dashboard, ensuring situational awareness of the worst-case scenario.
- ALERT (Not currently visible, but tracked): Logs high-severity events (Critical/Warning) that trigger the write_alert function, which simulates sending critical patient and geospatial data to HQ, fulfilling the seamless communication requirement.
